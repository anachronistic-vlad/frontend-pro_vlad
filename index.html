1) false && true || true --- true -> сперва выполняется &&, вторая команда запускается только в случае успешного завершения первой, первая неуспешная, значит false. Остаётся false || true, первая команда не выполняется, значит срабатывает вторая, а значит результат true.

false && (true || true); --- false -> результат в скобках равен true, остаётся false && true, так как первая часть не выполняется, то ко второй не доходит

+'40' + +'2'; --- 42 -> унарный плюс имеет приоритет выше чем бинарный, сперва происходит преобразование строк к числам и после уже сложение

'2' + 3 ** 2; --- "29"(строка 29) -> получаем строку, сперва происходит возведение в степень (3*3=9). Строка и число с бинарным плюсом это не сложение, а конкатенация, получаем строку "29"

3 ** 2 / 3; --- 3 -> a ** m/n = a ** m и находим корень n-ой степени.

3 ** (9 / 3); --- 27 -> сперва действие в скобках, после возведение в степень

'10' + (5 === 6); --- "10false" -> строгое сравнение даёт false, строка "10" + false = бинарный плюс приводит false к строке и происходит конкатенация строк

'10' - 5 === 6; --- false -> вычитание приводит строку к числу, получаем 5 === 6, а это false

undefined + 1; --- NaN -> любые операции с undefined дают NaN

(+null == false) (меньше) 1; --- false -> +null преобразовывается к 0, при сравнении 0 и false, false приравнивается к 0 и мы получаем true. true не меньше 1, значит получаем false

(+null == false) (меньше) 1 ** 5; --- false -> левая часть как и в прошлом примере получаем true, 1 в пятой степени это 1, значит снова false
